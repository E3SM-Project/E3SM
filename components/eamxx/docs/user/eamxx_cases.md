# EAMxx: Case basics and supported configurations

This page helps users create, configure, and run CIME cases that use EAMxx (SCREAM)
as the atmosphere component. It provides a practical, supported starting point and
lists commonly used/available compsets and grids. For advanced topics see the
linked pages below.

## Prerequisites

- Familiarity with CIME case workflow: create_newcase, case.setup, case.build,
  and case.submit. See the CIME user guide for general case control details.
- Access to a supported machine listed in the project documentation.

## Quick-start example

Create a simple debug case (adjust CASENAME, MACHINE):

```bash
cd cime/scripts
./create_newcase --case ../../CASENAME --compset F2010-SCREAMv1 --res ne4pg2_ne4pg2 --mach MACHINE
cd ../../CASENAME
./case.setup
./case.build
./case.submit
```

Use `./xmlquery EXEROOT` to locate executables and `./xmlquery RUNDIR` to find
the run directory.

## Supported EAMxx compsets (summary)

The EAMxx component exposes multiple compsets / configurations. Some are
production-like and tested, while others are experimental.

- F2010-SCREAMv1 — common SCREAM configuration for present-day climatology (2024+).
- F20TR-SCREAMv1 — time-varying historical forcing.
- F2010-SCREAMv1-noAero — aerosol-free variant.
- F2010-EAMxx-MAM4xx — EAMxx with the MAM4xx aerosol/optics package (aerosol-enabled).
- FIOP-SCREAMv1-DP / FRCE-SCREAMv1-DP — Doubly-periodic / IOP experimental configs (DP-EAMxx).
- DYAMOND / SCREAM-HR variants — high-resolution or DYAMOND experiment configs.
- ZM / EAMxx-LR development compsets — low-resolution variants that enable ZM deep-convection options.

Status notes
- Many of these compsets are marked experimental or under development in the
  component `cime_config`. Check `components/eamxx/cime_config/config_compsets.xml`
  for the authoritative list and current support_level.

## Common grids

- `ne4pg2_ne4pg2` — low-cost grid for development and smoke tests.
- `ne30pg2_ne30pg2` / `ne30pg2_r05_IcoswISC30E3r5` — higher-resolution grids used in
  production or performance studies.

Choose a grid appropriate to your machine and experiment goals.

## Enabling optional physics and aerosol packages

- MAM4xx (aerosols/optics) — available (config knobs in `namelist_defaults_eamxx.xml`);
  enabling MAM4xx changes available processes and may require additional input files.
- ZM deep-convection and GW (gravity-wave drag) — available as configurable
  atmosphere processes; refer to `namelist_defaults_eamxx.xml` for runtime knobs.

If you plan to enable MAM4xx, consult the developer testing pages (the component
has dedicated MAM4xx test suites) and ensure input files are available.

## Editing runtime parameters

EAMxx runtime parameters are generated by `buildnml` into `eamxx_input.yaml` and
`namelist.nl` in the run directory. Typical workflow:

1. Use `atmquery` to inspect parameter values:

```bash
./atmquery my_param
```

2. Use `atmchange` to update runtime parameters in the case (these edits are
   regenerated by `buildnml`, so modify through `atmchange` or case XMLs):

```bash
./atmchange homme::number_of_subcycles=10
```

Do NOT edit generated `eamxx_input.yaml` or `namelist.nl` by hand; they are
overwritten when `buildnml` runs.
